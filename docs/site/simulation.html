Simulation Framework

MHRQI runs on classical simulation using Qiskit Aer. Two simulation modes are supported depending on image size and accuracy requirements.

Simulation Modes

Statevector Simulation

Function: simulate_statevector()

Computes exact probability amplitudes. No sampling noise. Ideal for development and debugging.

✓ Exact results
✗ Memory intensive
✓ GPU acceleration available

Shot-Based Simulation

Function: simulate_counts()

Samples measurement outcomes over many "shots". More realistic model of quantum behavior.

✓ Realistic sampling
✓ Lower memory
✗ Statistical noise

Understanding Hits and Misses

After simulation, each measurement outcome includes the state of the outcome qubit. This determines whether the pixel is marked for smoothing.

Outcome Qubit Meaning Reconstruction Action
0 (Miss) Pixel is consistent with parent Preserve original intensity
1 (Hit) Pixel is inconsistent (likely noise) Smooth with neighbor median

The hit/miss ratio determines the confidence score used in image retrieval.

Complete Pipeline

graph LR A["MHRQI_init"] --> B["MHRQI_upload"] B --> C["DENOISER"] C --> D["simulate_statevector or simulate_counts"] D --> E["make_bins_sv or make_bins_counts"] E --> F["mhrqi_bins_to_image"]

The binning functions (make_bins_*) extract intensity and outcome statistics from measurement results, which are then used by the image retrieval algorithm.
